WePro upgraded v10

本次升级内容：
1. 增加启动页动画：Logo 缩放淡入、渐变背景、启动过渡。
2. 首页重新整理为卡片式布局，增加“无驱动模式 / 驱动模式”选择。
3. 无驱动模式：只需要选择运行文件，跳过驱动 ZIP、内核匹配和刷入步骤，直接准备并运行文件。
4. 驱动模式：需要选择驱动 ZIP，自动匹配当前内核版本，匹配成功后才能刷入并运行。
5. 修复二进制 main 信息不显示的问题：旧版使用后台 detached 启动，并把 stdout/stderr 重定向到 /dev/null；新版改为实时进程管道读取输出。
6. 增加终端输入框和发送按钮：可把卡密、确认内容、命令等写入正在运行进程的 stdin。
7. 增加停止按钮：可请求结束当前进程。
8. 替换系统文件选择器：新增自定义文件选择器，可浏览 /sdcard、根目录 /、上级目录。
9. 自定义文件选择器支持收藏当前目录，并会记住运行文件目录和驱动 ZIP 目录，便于下次快捷跳转。
10. 我的页面增加“授权访问全部文件”和“清空快捷目录”。

注意：
- 需要设备具备 root 权限，运行文件会复制到 /data 后 chmod 777 再运行。
- Android 高版本如果 /sdcard 目录显示不全，请在“我的 -> 授权访问全部文件”里给权限。
- 如果某些二进制强依赖 TTY/PTY 才刷新提示符，普通 stdin/stdout 管道可能仍然会出现提示不及时；当前版本已修复 stdout/stderr 被丢弃的问题，但没有引入第三方 PTY 库。
- 本环境没有 Android Gradle Wrapper jar / Android SDK，未重新编译 APK，提供的是完整源码工程。

## v11 升级记录

- 修复运行文件卡在“开始读取 stdout/stderr”但没有 main 输出的问题：运行流程改为交互式 root shell，并优先通过系统 `script` 命令创建 PTY 伪终端，二进制如果依赖 TTY 输出/输入时会更接近真实终端；设备没有 `script` 时自动回退普通管道。
- 运行前会显示调试日志：本地路径、Root 运行路径、root command dispatched、Root shell connected、终端模式、退出码，方便定位到底卡在复制、su、PTY 还是二进制本身。
- 运行目录从 `/data` 调整到 `/data/local/tmp/wepro_runner`，并在 root 下自动创建目录，降低 `/data` 直接执行/权限异常导致启动失败的概率。
- 停止按钮现在会先发送 Ctrl-C，再销毁 su 进程，并尝试按运行路径清理残留 root 进程。
- 首页/我的页面切换加入淡入滑动动画。
- 删除终端区域左上角“二进制终端输出”标题，界面更干净。
